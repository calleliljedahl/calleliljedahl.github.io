Robot doing dishes

Dogs are for old people

Having my coffee cups pile up after a day of work, and eager try something new, I bought a 3D-printer and started designing a robot with the goal of being able to pick up my dishes, and putting them in the dishwasher. I was facinated by the thought of being able to implement algorithms to a physical robot and see them in action. Today, over a year later - the robot is around 1mÂ³ in size, and able to do mapping and navigation using SLAM, walking around my apartment freely. At this point it is a part of the household family, and under constant developement. After implementing some interesting reinforcement learning algorithms to his cameras making him able to identify objects, I am now moving on to developing a deep learning implementation to the LIDAR-data to further improve his walking patterns. Having recently completed building a vertical elevator able to elevate above the kitchen's sink, I have started designing a six-joint arm for picking up dishes. The heart of the robot is linux based, using the the fantastic ROS2-framework allowing me to program in almost any language in separate repositories for the different nodes (I am using Python mainly, but C++, Javascript and C are used in different occurances - other than a lot of work in Unix shells).

Illustration shows how lidar data are used with SLAM to generate a 2D map. A map later used with deep learning to optimise path finding. A little demo of how the robot looks in action, aswell as showcasing PID-tuning the vertical elevator. Intitially it was run on tracks - but was changed to wheels with suspension due to irregular sensor-outputs caused by the bumpy tracks.
